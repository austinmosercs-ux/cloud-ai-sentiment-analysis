# AWS Setup Guide â€” AI Sentiment Analysis Project

Follow these steps in order. Everything uses the AWS Console (no CLI required).

## Step 1 â€” Create the DynamoDB Table

1. Open **DynamoDB** in the AWS Console.
2. Click **Create table**.
3. Fill in:
   - **Table name:** `SentimentReviews`
   - **Partition key:** `reviewId` (type: **String**)
4. Leave all other settings as default (On-demand capacity is fine for a project).
5. Click **Create table**.

**Table Schema:**

| Attribute    | Type   | Notes                              |
|-------------|--------|------------------------------------|
| reviewId    | String | Partition key â€” UUID generated by Lambda |
| timestamp   | String | ISO 8601 UTC timestamp              |
| reviewText  | String | The raw review text                 |
| sentiment   | String | POSITIVE / NEGATIVE / NEUTRAL / MIXED |
| scores      | Map    | {positive, negative, neutral, mixed} as floats |

---

## Step 2 â€” Create the IAM Role for Lambda

1. Open **IAM** â†’ **Roles** â†’ **Create role**.
2. Choose **AWS service** â†’ **Lambda** â†’ Next.
3. Add these managed policies:
   - `AWSLambdaBasicExecutionRole` (CloudWatch logging)
   - `ComprehendReadOnly` (DetectSentiment)
   - `AmazonDynamoDBFullAccess` (read/write SentimentReviews table)

   > For production you would scope DynamoDB to just one table with a custom policy,
   > but FullAccess is fine for a class project.

4. Name the role: `LambdaSentimentRole`
5. Click **Create role**.

---

## Step 3 â€” Create the Lambda Function

1. Open **Lambda** â†’ **Create function**.
2. Choose **Author from scratch**.
3. Fill in:
   - **Function name:** `SentimentAnalysisFunction`
   - **Runtime:** `Python 3.12`
   - **Architecture:** `x86_64`
   - **Permissions:** Use existing role â†’ `LambdaSentimentRole`
4. Click **Create function**.

### Upload the code

1. In the Lambda console, click the **Code** tab.
2. Click **Upload from** â†’ **.zip file** OR paste the code directly in the inline editor.
   - If pasting: delete the existing `lambda_function.py` content and paste all of `lambda/lambda_function.py`.
3. Click **Deploy**.

### Configure the environment

1. Click the **Configuration** tab â†’ **General configuration** â†’ **Edit**.
2. Set **Timeout** to `30 seconds` (Comprehend API can take a moment).
3. Click **Save**.

---

## Step 4 â€” Create the API Gateway

1. Open **API Gateway** â†’ **Create API**.
2. Choose **HTTP API** â†’ **Build**.
3. Click **Add integration** â†’ **Lambda** â†’ select `SentimentAnalysisFunction`.
4. **API name:** `SentimentAPI`
5. Click **Next**.

### Add routes

On the **Configure routes** screen:

| Method | Path      |
|--------|-----------|
| POST   | /analyze  |
| GET    | /history  |

For both routes, set the integration to your Lambda function.

6. Click **Next** â†’ **Next** â†’ **Create**.

### Get your API URL

After creation, copy the **Invoke URL** â€” it looks like:
```
https://abc123xyz.execute-api.us-east-1.amazonaws.com
```

## Step 5 â€” Enable CORS on API Gateway

1. In API Gateway, click your API â†’ **CORS**.
2. Fill in:
   - **Access-Control-Allow-Origin:** `*`  (use your S3 URL in production)
   - **Access-Control-Allow-Headers:** `Content-Type`
   - **Access-Control-Allow-Methods:** `GET, POST, OPTIONS`
3. Click **Save**.

## Step 7 â€” Host the Frontend on S3

1. Open **S3** â†’ **Create bucket**.
2. Fill in:
   - **Bucket name:** `sentiment-analysis-frontend-YOURNAME` (must be globally unique)
   - **Region:** same as your Lambda (e.g. `us-east-1`)
3. **Uncheck** "Block all public access" (required for static hosting).
4. Acknowledge the warning and create the bucket.

### Enable static website hosting

1. Click your bucket â†’ **Properties** tab.
2. Scroll to **Static website hosting** â†’ **Edit**.
3. Enable it, set **Index document** to `index.html`.
4. Save changes.

### Upload the file

1. Go to the **Objects** tab â†’ **Upload** â†’ add `index.html` â†’ Upload.

### Set bucket policy (make it public)

1. Go to the **Permissions** tab â†’ **Bucket policy** â†’ **Edit**.
2. Paste this policy (replace `YOUR-BUCKET-NAME`):

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME/*"
    }
  ]
}
```

3. Click **Save changes**.

### Get your website URL

In the **Properties** tab â†’ **Static website hosting**, copy the **Bucket website endpoint**.
It will look like:
```
http://sentiment-analysis-frontend-YOURNAME.s3-website-us-east-1.amazonaws.com
```

## Troubleshooting

| Problem | Fix |
|--------|-----|
| Lambda returns 500 | Check CloudWatch Logs (Lambda â†’ Monitor â†’ View CloudWatch Logs) |
| CORS error in browser | Make sure API Gateway CORS is saved and re-deployed |
| "AccessDeniedException" from Comprehend | Check the IAM role has `ComprehendReadOnly` |
| DynamoDB "ResourceNotFoundException" | Verify table name is exactly `SentimentReviews` |
| Frontend shows old API URL | Hard-refresh the browser (Cmd+Shift+R) after re-uploading to S3 |
